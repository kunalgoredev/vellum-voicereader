## What Was Built This Session

### 1. Live Reading (SSE streaming)
Full chunk-by-chunk live playback while generation happens in parallel.

**Backend ‚Äî 3 new endpoints in `app/main.py`:**
- `POST /api/generate/enqueue` ‚Äî creates job, chunks text, returns `{job_id, total_chunks}` instantly
- `GET /api/jobs/{job_id}/stream` ‚Äî SSE stream; generates one chunk at a time via `asyncio.to_thread`, yields `event: chunk` per chunk, `event: done` at end
- `GET /api/jobs/{job_id}/chunks/{idx}` ‚Äî serves individual WAV chunks (0-indexed)

**Frontend ‚Äî `LivePlayer` class in `web/app.js`:**
- Opens `EventSource` to the SSE stream
- `addChunk(idx, url)` queues audio; starts playing on chunk 0, auto-advances on `ended`
- `waitingForChunk` flag handles generation being slower than playback
- Pause/Resume/Stop controls

### 2. Full UI Redesign

- **Dark premium theme** ‚Äî CSS variables, subtle surface layers, glow shadows on primary button

### 3. Electron App
**Files created:**
- `electron/main.js` ‚Äî main process: setup wizard + server manager
- `electron/preload.js` ‚Äî context bridge for splash IPC
- `electron/splash.html` ‚Äî dark loading screen with animated progress bar
- `package.json` ‚Äî electron-builder config (Windows NSIS + Mac DMG)

**Electron flow:**
1. Show splash window
2. `findUv()` / `findSystemPython()` ‚Äî locate tooling without any shell spawn
3. Create `.venv` in `%APPDATA%\Vellum\` (userData) via uv or pip
4. `pip install -r requirements.txt`
5. Download Kokoro model (~310 MB) via `scripts/download_kokoro_model.py`
6. Start uvicorn on port 8000
7. Poll until server responds ‚Üí open main BrowserWindow

### 4. Apple Silicon (MPS) Support ‚Äî `app/tts_engine.py`
- `_best_device()` ‚Üí CUDA ‚Üí MPS ‚Üí CPU priority
- `_free_device_cache()` calls `torch.mps.empty_cache()` after each chunk
- `torch.inference_mode()` wraps generation (lower RAM)
- Device fallback: if MPS fails on a specific op, drops to CPU silently

### 5. Path changes ‚Äî `app/config.py` + `scripts/download_kokoro_model.py`

## Errors Encountered This Session (Electron)

### Error 1 ‚Äî `spawn C:\Users\Kunal\.local\bin\uv.exe ENOENT`
**Cause:** `findBin()` used `execSync('where uv')` which returned a path that existed in shell PATH but the file didn't exist on disk.  
**Fix attempt:** Replaced with `spawnSync('uv', ['--version'], { shell: IS_WIN })` ‚Äî returned `'uv'` as command name.

### Error 2 ‚Äî `spawn uv ENOENT`
**Cause:** `runProc` used `shell: IS_WIN` (true on Windows). This worked in `_tryCmd` but when spawning `'uv'` without a full path and with `shell: true`, Node.js tries to invoke `cmd.exe` internally.  
**Fix attempt:** Changed `shell: IS_WIN` to `shell: false` in `runProc`.

### Error 3 ‚Äî `spawn C:\Windows\system32\cmd.exe ENOENT`
**Cause:** `shell: true` in `runProc` caused Node to try to spawn `cmd.exe` to resolve `'uv'`. Packaged Electron blocks `cmd.exe` spawning in some configurations.  
**Fix attempt:** Removed all `shell: true`. Rewrote `findUv`/`findSystemPython` to walk `process.env.PATH` directly in Node.js (`_searchPath`), no subprocess needed.

### Error 4 ‚Äî `spawn C:\Users\Kunal\.local\bin\uv.EXE ENOENT` (still failing as of session end)
**Cause:** `_searchPath` found `uv.EXE` in PATH via `fs.existsSync` ‚Äî the file appears in directory listing but cannot actually be executed (broken/stale uv install, possibly from WSL symlink or corrupted install). `fs.existsSync` returns `true` but `spawn` returns ENOENT.  
**Fix applied (NOT YET TESTED):** Added `_canRun(fullPath)` using `execFileSync(path, ['--version'], { stdio: 'pipe' })` ‚Äî no shell, direct binary probe. Both `fs.existsSync` AND `_canRun` must pass. Broken uv is skipped, falls through to pip.
